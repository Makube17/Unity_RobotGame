# AIキャッチゲーム

## 概要 / Overview

ランダムに積み重なった物体に対して、AIを用いて把持候補推定を行い、プレイヤーが候補を選択してロボットハンドで把持を行うUnityゲームデモです。

## 制作目的 / Purpose

自身の研究テーマである、RGB-D画像および深度画像を用いた物体把持システム、ゲーム形式で分かりやすく体験できる形に落とし込むことを目的としています。
スキャン、把持候補の表示、候補選択、ロボットハンドによる把持動作までを一連のインタラクションとして実装しています。

## 使用技術

- Unity
- C#
- Robot hand simulation
- FastSAM for Calibration
- Dex-Net for grasp candidate estimation
- HTTP / JSON communication with external AI server
- Game state management

## システム構成

Unity側では、物体生成、スキャン操作、把持候補の表示、候補選択、ロボットハンドの動作、成功・失敗判定を担当します。
AIサーバー側では、RGB-D画像および深度画像を受け取り、FastSAMによるセグメンテーションとDex-Netによる把持位置推定を通して、把持パラメータをUnityへ返します。

処理の流れは以下の通りです。

- Unity上で物体をランダム生成
- RGB-D画像と深度画像を取得
- AIサーバーへ画像を送信
- FastSAM、Dex-Netにより把持候補を推定
- Unity上に把持候補マーカーを表示
- プレイヤーが候補を選択
- 把持位置をロボットハンド座標系へ変換
- ロボットハンドが把持動作を実行
- 把持成功・失敗を判定

## 機能

- ランダムな物体生成
- 物体生成中・スキャン中の操作ロック
- RGB-D画像の送信
- Dex-Netによる把持候補の取得
- 把持候補マーカーの表示
- プレイヤーによる候補選択
- Unity座標系からロボットハンド座標系への変換
- ロボットハンドによる把持動作
- 簡易的な成功・失敗判定

## 主なスクリプト

- `ScanManager.cs`  
  スキャン処理、AIサーバーとの通信、把持候補の表示、候補選択、成功判定などを管理します。

- `RobotHandController.cs`  
  ロボットハンドの回転、把持動作、固定ベース時の座標変換・角度計算を管理します。

- `ImageSender.cs`  
  画像や関連データをAIサーバーへ送信する処理を担当します。

- `ObjectSpawner.cs`  
  物体のランダム生成を担当します。

- `SpawnedObjectPhysicsStabilizer.cs`  
  生成された物体の物理挙動を調整し、デモ中に物体が過度に弾かれにくくするための補助スクリプトです。

## 工夫点

- `GameState` によってゲーム進行を管理し、物体生成中やスキャン中に不適切な操作ができないようにしました。
- 把持候補をマーカーとして表示し、プレイヤーが直感的に候補を選択できるようにしました。
- Unity座標系とロボットハンド座標系の対応関係を整理し、把持位置をハンド基準の座標へ変換できるようにしました。
- 研究システムをそのまま再現するのではなく、ゲームデモとして遊べる形に簡略化しました。
